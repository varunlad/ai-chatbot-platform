# Authentication

The application uses **JWT authentication with HTTP-only cookies**.

### Authentication Flow

1. User signs up.
2. User logs in.
3. The backend generates a JWT.
4. The JWT is stored in a secure **HTTP-only cookie**.
5. The browser automatically sends the cookie with every subsequent request.
6. The frontend never stores or manages the JWT.

# 1. User Signup

Creates a new user account.

## Endpoint

```
POST /auth/signup
```

## Request Body

```json
{
  "name": "Varun",
  "email": "varun@email.com",
  "password": "Sujata@10"
}
```

```json
{
  "success": true,

    "email": "varun@email.com",
    "createdAt": "2026-07-19T12:00:00.000Z",
    "updatedAt": "2026-07-19T12:00:00.000Z"
```

# 6. Send Chat Message

Sends a message to the selected AI assistant.

## Endpoint

```
POST /chat
```

## Authentication

Required (HTTP-only Cookie)

## Request Body

```json
{
  "conversationId": "conversationId",
  "message": "What is React?"
}
```

## Backend Processing

When this endpoint is called, the backend performs the following steps:

1. Validates the authenticated user.
2. Verifies conversation ownership.
3. Stores the user's message.
4. Loads complete conversation history.
5. Reads the selected assistant type.
6. Loads the corresponding system prompt.
7. Sends the conversation history to OpenRouter.
8. Receives the AI response.
9. Stores the AI response.
10. Returns the AI response to the client.

## Success Response

```json
{
  "success": true,
  "data": {
    "conversationId": "conversationId",
    "messageId": "messageId",
    "response": "React is a JavaScript library for building user interfaces."
  }
}
```
